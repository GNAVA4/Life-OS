# Life OS

Персональный лайф‑трекер: задачи, привычки, цели, дела, заметки и финансы в одном приложении, с геймификацией (XP, уровень, стрик, здоровье, ⚡импульс, достижения). Работает как веб‑приложение и как Android‑приложение (одна кодовая база).

> Личный проект для ежедневного планирования и рефлексии. Данные хранятся локально в браузере/приложении и по желанию синхронизируются между устройствами через Firebase.

---

## Возможности

- **Сегодня** — задачи дня (со сложностью→XP), ежедневные и многодневные дела, теги, оценка дня, сон, заметка; перенос невыполненных задач со вчера; шаблоны наборов задач.
- **Привычки** — расписание (каждый день / дни недели), сгорающий стрик, заморозки, челленджи на N дней, напоминания; архив завершённых привычек.
- **Цели** — год/месяц/неделя/день; трекеры: ползунок / чек‑лист / счётчик; периодизация с архивом и rollover; мульти‑привязка задач и привычек к нескольким целям.
- **Дела** — эпик‑группировка, статусы, важность/срочность, дедлайны с подсветкой, архив.
- **Заметки** — заметки и напоминания (с повтором), закрепление, чек‑листы.
- **Финансы** — операции, счета и замеры, должники, планируемые расходы/доходы по месяцам, бюджет‑алерты с прогнозом, графики.
- **Статистика** — дисциплин‑грид, обзор периода, тренды, план/факт, анализ факторов оценки дня (корреляции).
- **Геймификация** — XP и уровень, стрик, здоровье, ⚡импульс, ~286 достижений.
- **Уведомления** (Android) — привычки, напоминания, дедлайны, утренняя сводка.
- **Бэкап** — экспорт/импорт JSON (полный), экспорт в Excel; на телефоне — через системное «Поделиться». Синхронизация через Firebase Firestore (по желанию, вход через Google).

## Стек

| Слой | Технология |
|------|-----------|
| UI | React 18 |
| Сборка | Vite 5 |
| Графики | Chart.js 4 |
| Экспорт | SheetJS (xlsx) |
| Хранилище | `localStorage` (ключи `lifeos:*`) |
| Синхронизация | Firebase Firestore (last‑write‑wins по ключу) |
| Android | Capacitor 8 (local‑notifications, share, filesystem, firebase‑authentication) |

## Запуск и сборка

### 1. Настройка Firebase (нужна для синхронизации)

Синхронизация опциональна, но конфиг нужен для сборки. Создай проект в [Firebase Console](https://console.firebase.google.com/), включи **Firestore** и **Authentication → Google**, затем:

```bash
cd app/src
cp firebaseConfig.example.js firebaseConfig.js   # и впиши свои значения из Firebase (Web app config)
```

Для Android‑входа через Google положи `google-services.json` (из Firebase) в `app/android/app/`.
Оба файла в `.gitignore` и в репозиторий не попадают.

### 2. Веб (разработка)

```bash
cd app
npm install
npm run dev        # локальный dev‑сервер
npm run build      # прод‑сборка в app/dist
```

### 3. Android (.apk)

Требуется Android SDK + JDK (проще всего через Android Studio; путь к SDK в `app/android/local.properties`).

```bash
cd app
npm run android:sync                 # vite build + cap sync android
cd android
./gradlew.bat assembleDebug          # Windows (PowerShell); на macOS/Linux: ./gradlew assembleDebug
```

Готовый APK: `app/android/app/build/outputs/apk/debug/app-debug.apk`.

Открыть проект в Android Studio: `npm run android:open`.

## Данные и приватность

- Все данные лежат в `localStorage` под ключами `lifeos:*` (задачи, финансы, привычки и т.д.) — на устройстве/в профиле браузера.
- Синхронизация включается входом через Google; данные пишутся в Firestore проекта, доступ ограничивается правилами Firestore.
- Полный бэкап — экспорт в JSON (все ключи `lifeos:*`); импорт восстанавливает их.
- Ключи Firebase и `google-services.json` не хранятся в репозитории (см. `.gitignore`).
